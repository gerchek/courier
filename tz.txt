###Register и Forgot your password?  Удаляем. 

###У Admin полные права, может добавлять/регистрировать/удалять любую из групп в том числе нового админа.
Admin может удалять/добавлять посылки и курьеров,полный контроль везде, а так же менять данные "edit"

Support все тоже самое, кроме регистрации/добавления юзеров, может добавлять/регистрировать только курьеров и по остальным параметрам тоже полный доступ,редактирование итд
Support видит всех курьеров, все посылки и юзеров

Users доступ только к вкладке "курьеры", которые были добавлены к данному юзеру из общего списка. И доступ к вкладке посылки,в этой вкладке users видят только свои добавленные посылки, общий список/чужие посылки других юзеров/админов/supporta не видят

Users не могут редактировать ничего кроме добавленной посылки. Например "edit" и внес описание товара,если было пустое или неполное.

Группы admin/support/users входят только по логину и паролю, который им выдал admin, email они не используют
Admin может редактировать их пароль в случае чего.
При добавлении users или курьера в форму ввода "login" должно разрешаться вписывать любые значения логином может является просто почта email@email.com ,но только как вид логина, письма никуда не идут.
Для курьеров проверка пароля должна быть упращенной так чтобы любой легкий пароль работал по типу 12345.

Чтобы упростить задачу можно сделать одну вкладку слева Packages - admin/support видят абсолютно все добавленные посылки 
Users будут видеть там только те посылки,которые они добавили. (курьеры в другой группе и у них своя страница логина)

3) Страницы Admin/Support

Так же справа есть общий Packages они видят все добавленные посылки от юзеров итд и от себя тоже. 

Во вкладке посылки для них доступно редактировать/ удалит посылку и search bar например по номеру трека или id, 
например номер отслеживания послыки 1234567 , ввел 67 и он показывает или полный трек либо номер id

Цвет статуса тоже передается в эту вкладку packages. Если курьер нажал Task Received, в этой вкладке видно синий например, когда Package Received нажали, то видно зеленый 

Сам раздел тоже таблица например  ID , Courier , Status , Name on the box, Description,  Tracking number ,  Shipping Label

Shipper - имя юзера, кто добавлял посылку или admin/support/user 
id - id посылки
courier - имя курьера
status Task Recevied/Package Received/Package shipped    меняется в зависимости от того что нажал курьер
Name on the box - John Doe   отображает имя на посылке
Description  описание посылки просто текст
Tracking number  номер отслеживания Fedex/UPS/USPS
Shipping label  тоже номер отслеживания только новый исходящий,он будет от залитого файла PDF. 1234567.PDF  и показывает 1234567

При редактировании посылки в данной вкладке admin/support могут загрузить файл PDF 
например нажали edit и видят:
Courier 
User: имя юзера или админ 
Description
Name on the box
Shipping company  USPS/UPS/Fedex
Tracking Number
Package Status
Shipping Label upload file и сюда сразу можно залить PDF лейбл 

обычные users так же могут загрузить файл лейбл PDF,но только на свои добавленные посылки и общих они не видят.
